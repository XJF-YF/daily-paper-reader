## 摘要
三维高斯泼溅（3DGS）被广泛用于捕捉和渲染真实场景。将一个捕捉中的对象合成到另一个捕捉中在许多领域有应用，如视觉特效、建筑与室内设计或市场营销。然而，由于两个场景之间的光照条件不同，从源场景中提取对象并简单地粘贴到目标场景中无法产生逼真的结果。为了解决这个问题，我们引入了一种扩散模型，该模型可以调和光照不一致的简单合成图像。该模型使用异构的图像对数据集（不一致的合成输入，一致的输出）进行训练，结合了合成数据、生成数据和真实数据。我们的完整3D解决方案允许用户从源场景中提取对象并将其合成到目标场景中。由此，渲染出带有合成对象的目标场景的（不一致的）视图。我们的扩散模型对每个视图进行调和，最终通过后优化步骤将其整合到3DGS表示中。我们的方法提供了视觉上令人信服的结果，使得3DGS之间的对象迁移易于使用，并且与以前的方法相比显著提高了质量。

## Abstract
3D Gaussian Splatting (3DGS) is widely used to capture and render real scenes. Compositing objects from one capture into another has applications in many domains, such as VFX, architecture and interior design, or marketing. However, extracting an object from a source scene and naively pasting it into a target scene will fail to produce realistic results due to the different lighting conditions between the two scenes. To address this problem, we introduce a diffusion model that harmonizes naively composited images with inconsistent lighting. The model is trained with a heterogeneous dataset of image pairs (inconsistent composite input, consistent output), combining synthetic, generated, and real data. Our complete 3D solution allows a user to extract an object from the source scene and composite it into the target scene. From this, the (inconsistent) views of the target scene with the composite object are rendered. Our diffusion model harmonizes each one of these views, which are finally consolidated in a 3DGS representation with a post-optimization step. Our method provides visually compelling results, making object transfer between 3DGS easy to use and significantly improving quality compared to previous methods.

- **应用背景**：视觉特效、建筑可视化、室内设计、市场营销等领域需要从不同捕获中复用资产，但传统方法依赖专业建模和光照估计，成本高昂。现有2D调和扩散模型缺乏3D一致性，而3D逆渲染方法无法处理交互阴影和前景-背景相互作用。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 提出 **DOT3D（Diffusion Object Transfer in 3D）**，一个完整3D对象迁移管线，包含三个主要步骤：  
  1. **对象提取与交互式放置**：从源场景中分割对象，用户可在目标场景中调整位置、旋转和缩放。  
  2. **单视图光照调和**：使用微调的扩散模型对每张合成视图进行光照调和，产生与目标场景光照匹配的图像（包括阴影和反射）。  
  3. **多视图后优化**：将独立调和的视图通过3DGS后优化步骤整合为多视图一致的场景。

### 关键技术细节
- **对象提取**：基于3DGS，为每个高斯基元添加一个二进制特征维度，通过2D掩膜监督训练交叉熵损失，用阈值0.75分割对象。
- **交互式接口**：允许用户加载提取的对象，在目标场景中调整位置/方向/缩放，并裁剪边界框去除伪影。
- **单视图调和模型**：基于FLUX.1-schnell（整流流模型）微调。输入为**合成图像**（对象被不正确照明的粘贴结果）和**对象掩膜**，输出调和后的图像。损失函数为整流流匹配损失：
  \[
  L_{\theta} = \mathbb{E}\left[\|v_{\theta}(z_t, z_{\text{comp}}, z_{\text{mask}}, c, t) - (\epsilon - z_0)\|_2^2\right]
  \]
  其中 $z_t$ 为噪声潜变量，$z_{\text{comp}}$ 和 $z_{\text{mask}}$ 为合成图像和掩膜的潜变量（不添加噪声），$v_{\theta}$ 预测速度场。
- **训练数据构建**：需要成对数据（不一致输入，一致目标）。通过三种数据源生成：
  - **合成数据**：Blender场景，渲染对象在不同环境光下的孤立图像，粘贴到背景上形成不一致输入，场景本身渲染作为一致目标。
  - **生成数据**：用FLUX生成图像，用BiRefNet获取掩膜，用自训练的对象移除网络获得背景，用DiffusionRenderer重新照亮对象。
  - **真实数据**：ORIDa数据集，同样用DiffusionRenderer重照。
- **多视图整合**：初始化3DGS为目标场景加上对象高斯基元，只优化颜色（所有球谐系数），保持几何和透明度不变。使用**感知损失（LPIPS）** 替代光度损失，避免跨视图不一致；关闭密化和透明度重置。

### 算法流程（文字说明）
1. 重建源场景3DGS，提取对象高斯基元。
2. 用户交互放置对象到目标场景，得到合成场景。
3. 对合成场景的每个视角渲染，调用调和扩散模型（运行4次去噪步）生成调和图像。
4. 用调和后的图像作为监督，对合成场景的3DGS进行后优化（仅颜色，感知损失，10K迭代）。
5. 输出多视图一致的调和场景。
